PHiLe

The Painless High-Level Persistence Engine

What's PHiLe?

PHiLe (pronounced fillet), the Painless High-Level Persistence Engine, is a next-generation database management tool and domain-specific language. It allows application developers to write what is basically a textual form of a conceptual or entity-relationship model, from which it automatically generates a Database Abstraction Layer for a variety of programming languages and database engines.

Unlike traditional ORMs, PHiLe has a strong, expressive type system. It allows programmers to work with many kinds of values, not only objects, therefore trivially eliminating many of the bottlenecks commonly associated with ORM solutions, such as having to retrieve the transitive closure of a list of objects only to discard all but a few of their attributes.

How does it work?

The DSL used by PHiLe is a statically-typed, object-oriented, functional-declarative language, which is capable of describing schemas (like a DDL) and queries/updates (like a DML). It brings the memory model programmers are familiar with to the world of persistent storage: relational tables become classes or structs, records and documents become objects, and relationships become arrays and pointers.

As rightfully expected from a modern functional language, the PHiLe DSL supports advanced and convenient features such as higher-order functions, pattern matching, and algebraic data types (including class, struct, tuple and enum types). The presence of enums, in particular, allows for type-safe data modeling in lieu of stringly-typed APIs, while proper optional types prevent errors related to NULL values.

PHiLe will support the following database flavors and programming languages:

Database Engines:

  • SQLite3
  • MySQL / MariaDB
  • MongoDB

Programming Languages

  • Rust
  • C
  • C++
  • Objective-C
  • Swift
  • Python
  • Java
  • Go
  • JavaScript

Support for other popular storage engines (e.g. Postgres, MS/SQL, Redis, …) and programming languages (e.g. Haskell, Ruby, C#, …) would be nice and is planned.
